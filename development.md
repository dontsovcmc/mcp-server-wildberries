# Разработка и диагностика

## Правила разработки MCP-серверов

### Архитектура

- **Search + Execute при 30+ действиях.** Не регистрировать сотни `@mcp.tool()`. Три инструмента (`wb_search`, `wb_execute`, `wb_execute_file`) экономят токены в контексте LLM — схемы 3 инструментов вместо сотен.
- **Singleton API-клиент.** Кешировать экземпляр HTTP-клиента, не создавать на каждый вызов.
- **Server instructions.** Передавать `instructions=` в FastMCP, чтобы LLM знал как использовать сервер (какие инструменты вызывать первыми, как искать действия).

### Параметры и модели

- **LLM никогда не угадывает параметры.** Каждое действие имеет Pydantic-модель с `Field(description=...)`. LLM видит полную JSON Schema: обязательные поля, типы, допустимые значения, дефолты.
- **Модели строятся по swagger-спецификациям.** Не по догадкам. Скачать yaml/json спецификацию API, прочитать схемы запросов, создать точные модели.
- **Никаких PassThroughParams.** Если действие принимает параметры — для него должна быть конкретная Pydantic-модель. Пустая модель с `extra="allow"` допустима только если swagger-спецификация недоступна.

### Именование

- **Единые ID везде.** MCP action ID = CLI команда = ID в документации. Один kebab-case формат (`fbs-order-cancel`), никаких маппингов между разными схемами именования.
- **CLI — фиксированные команды.** Каждое действие = отдельный subparser с `--help` и описанием аргументов. CLI не использует search/execute — это паттерн для LLM.

### Документация

- **Документация per-domain.** Отдельный MD-файл на каждый раздел API со списком всех команд, параметров и флагов. Ссылки из главного README.
- **README для не-программистов.** Пошаговая установка с разделением macOS/Linux/Windows, примеры команд с реальным выводом. Разные способы подключения (pip, uvx, --mcp-config).
- **Ссылка на официальную документацию API.** README указывает, что сервер построен по официальным swagger-спецификациям.

### Обработка данных

- **Валидация путей файлов** — через `_safe_output_path()`. Разрешены только домашняя директория и системный temp. Запись в dotfiles/dotdirs под home запрещена (`~/.ssh`, `~/.bashrc`, `~/.config/...`). Пути разворачивать через `os.path.realpath()` (защита от `..` и симлинков).
- **JSON от пользователя** — только через `_parse_json(text, label)`, не голый `json.loads()`. Пользователь должен получить понятное сообщение об ошибке, а не стектрейс.
- **Не включать `resp.text` в исключения.** Тело ответа API может содержать персональные данные. В исключение — только метод, путь и статус-код: `raise RuntimeError(f"GET {path} -> {resp.status_code}")`. Полный ответ логировать только на уровне `log.debug()`.

### Обработка ошибок

- **Не глотать исключения молча.** `except Exception:` без логирования запрещён. Всегда логировать через `log.warning()` или `log.error()` с контекстом.
- **HTTP-ошибки** — `wb_api.py` бросает `RuntimeError` с методом, путём и статус-кодом (без тела ответа). Не перехватывать в tool-функциях — FastMCP сам вернёт ошибку клиенту.

### Стиль кода

- **Читаемые имена хелперов** — `_to_json`, `_parse_json`, `_safe_output_path`, а не `_j`, `_p`, `_s`.
- **Типы** — аннотировать параметры и возвращаемые значения. Использовать встроенные типы Python 3.10+ (`dict`, `list`, `str`), не `Dict`, `List`.
- **HTTP через хелперы** — `_get()`, `_post()`, `_put()`, `_delete()`, `_get_bytes()`. Не ручные `session.get()`.
- **HTTP таймауты** — 30 секунд на обычные запросы, 60 секунд на файловые операции.
- **stdout зарезервирован** для JSON-RPC. Весь логгинг — только через `logging` в stderr.
- **Линтер** — `ruff check src/ tests/` должен проходить без ошибок перед коммитом. Максимальная длина строки — 120 символов.

### Безопасность

- **Проверка персональных данных перед коммитом.** Grep на токены, ИНН, имена, email в коде, тестах и документации. Заменять на вымышленные.
- **Секреты** — только через переменные окружения (`os.getenv`). Не хардкодить токены в коде или тестах. В тестах использовать `test-fake-token`.

## Как Claude запускает MCP-сервер

Claude Code запускает MCP-сервер как **дочерний процесс** (subprocess) и общается с ним по **stdin/stdout** через JSON-RPC 2.0:

```
Claude Code                          MCP-сервер
    │                                    │
    │── spawn subprocess ──────────────►│  (command + args из конфига)
    │                                    │
    │── stdin: {"method":"initialize"} ─►│  handshake (таймаут 10 сек)
    │◄─ stdout: {"capabilities":...} ───│
    │                                    │
    │── stdin: {"method":"tools/list"} ─►│  получить список инструментов
    │◄─ stdout: [tools...] ─────────────│
    │                                    │
    │        ✓ Connected                 │
```

**Важно:**
- Сервер **не запускается в login shell** — `~/.bashrc`, `~/.zshrc` не выполняются
- **PATH может быть урезан** — команда может не находиться, хотя в обычном терминале работает
- **stdout зарезервирован** для протокола — любой `print()` в stdout ломает соединение (используйте stderr)

## `✗ Failed to connect` — что делать

**1. Запустить сервер вручную:**

```bash
uvx mcp-server-wildberries
# или
mcp-server-wildberries
```

**2. Проверить что команда доступна:**

```bash
which mcp-server-wildberries
```

**3. Использовать абсолютный путь** (если PATH урезан):

```bash
which mcp-server-wildberries
# Использовать полный путь в конфиге
```

**4. Посмотреть логи Claude Code:**

```bash
# macOS:
tail -f ~/Library/Logs/Claude/mcp*.log
# Linux:
ls ~/.config/Claude/logs/
```

**5. Включить debug-режим:**

```bash
claude --mcp-debug
```

## Частые проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| `command not found` | Пакет не установлен | `pip install mcp-server-wildberries` |
| `Failed to connect` | PATH урезан | Используйте абсолютный путь |
| `401 Unauthorized` | Неверный API-токен | Проверьте `WB_TOKEN` |
| `403 Forbidden` | Недостаточно прав токена | Создайте токен с нужными правами в ЛК WB |
| `No module named` | Другой Python | Используйте entry point вместо `python -m` |
| `429 Too Many Requests` | Превышен лимит запросов | Content: 100 req/min, Marketplace: 300 req/min |
